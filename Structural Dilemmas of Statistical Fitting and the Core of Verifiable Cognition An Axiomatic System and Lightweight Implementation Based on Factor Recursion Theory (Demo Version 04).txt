摘要
当前大规模统计学习系统在实际应用中存在“幻觉”现象，即系统在缺乏充分证据时生成貌似合理但事实上不成立的陈述。本文指出，这一问题不仅是工程调优不足的结果，更与统计主导范式在可验证性与责任追溯结构上的内在特征相关。
为此，本文提出“因子递归论”作为一个形式化公理系统。该系统由五条公理构成：实时交互、结构解耦、协议约束、可计算可追溯、功率效率约束。与“生成—外部校验”模式不同，该系统在架构层面对合法操作施加内生约束，使系统输出仅来自可重建的推理路径。
本文进一步实现了一个轻量化逻辑推理引擎原型（v0.3.1），其核心基于SLD解析，辅以能量消耗模型与可追溯记录结构。演示版代码公开，展示“无证据不输出”的基本实现机制。
本文不主张替代统计方法，而提出一种可作为高可靠性场景安全内核的形式化认知单元。
关键词：统计拟合；幻觉；形式化推理；因子递归论；可验证认知；能量约束；可追溯系统

引言
近年来，深度学习模型在自然语言处理、计算机视觉等领域取得显著成果。然而，在开放环境中，这类模型常出现以下现象：
•	编造不存在的事实；
•	构造不可验证的引用；
•	在低置信信息条件下给出高置信表达。
尽管检索增强生成（RAG）、工具调用与符号模块被引入以缓解问题，但其核心生成机制仍依赖统计拟合函数。统计模型的输出并非基于形式化推导，而是对训练分布的条件概率近似。
本文认为：在需要强可验证性与责任可追溯的场景中（如法律辅助、数学证明、关键基础设施控制），统计范式存在结构性可验证性赤字。为此，我们构建一个形式系统，其合法性定义为：系统输出必须来源于形式公理与显式可重建推理链的闭包。

第一部分：统计主导范式的结构性困境
1.1 理论收敛与现实偏离
统计学习理论在理想条件下具有渐近一致性：当训练数据无限且独立同分布时，模型可以逼近真实分布。然而，现实世界永远无法满足这些条件——样本有限、分布随时漂移、对抗输入存在。因此，模型输出与物理实在之间必然存在不可控的偏离。这不是理论缺陷，而是工程常态。
1.2 推理过程不可显式验证
当前主流架构缺乏对推理过程的显式验证机制。典型的深度神经网络本质上是一个巨大的函数逼近器，其内部表示无法直接解释，决策路径无法审计。即使模型在测试集上准确率高达99.9%，那0.1%的错误仍可能在关键系统中造成灾难，且错误发生时无法追溯推理路径、无法界定责任主体。
1.3 资源假设与物理成本
统计模型训练和推理消耗巨额计算资源，但这一消耗几乎没有物理意义上的约束。模型可以无限搜索、无限迭代，直至找到符合统计模式的答案。这种“无限资源假设”不仅不经济，更掩盖了认知过程本应具有的物理代价——兰道尔原理告诉我们，任何信息处理都必须支付能量。一个认知系统若要具备物理一致性，应在设计层面体现资源约束。
1.4 混合架构的贡献与局限
检索增强、工具调用、符号模块等混合架构显著提升了系统的可靠性，但它们往往是黑箱的“插件”：检索可能召回错误文档，工具调用可能依赖不可靠API，符号约束可能在复杂推理中失效。更重要的是，它们缺乏一个统一的可验证推理核心，整个系统的决策过程仍然是碎片化的，难以端到端追溯。

第二部分：因子递归论——形式化公理系统
因子递归论从存在论层面重新定义了智能系统的基本单元，其五条公理构成任何可长期存活系统的充分必要条件。与“生成+验证”模式不同，这些公理是设计时嵌入的内生约束，非法操作在架构层面即被阻止。
2.1 公理1：实时交互——痕迹锚定
系统必须与物理世界维持持续的、双向的实时交互。交互本身留下痕迹，这些痕迹构成认知的物理锚点。当前原型中知识库为静态，预留了动态更新接口。
2.2 公理2：结构解耦——风险隔离
系统中生存、安全、分配、监督等核心模块必须保持结构隔离。这防止认知层的臆想直接传播到物理层。在推理引擎中，这体现为核心与插件的清晰边界。
2.3 公理3：协议约束——规则内嵌
所有涉及权力、资源、风险的交互必须基于可验证、可追责、可公开的规则。在推理层面，每一步推导都应遵循形式化的逻辑协议，而非黑箱的统计拟合。我们采用SLD解析作为基础逻辑协议。
2.4 公理4：可计算与可追溯——痕迹追溯
关键状态、贡献、成本必须可观测、可验证、可追溯。每个因子的状态都是历史交互的痕迹累积。引擎内置了TraceRecorder，记录每一步的知识库、因子、能量和决策。
2.5 公理5：功率效率——能量闭环
系统单位时间的总物质-能量-信息消耗必须低于环境可持续供给率。在推理中，每一步操作都有确定的能量代价，且总能量固定，能量耗尽即停止。当前原型中能量消耗是简单的计数器（每步+1），未与信息论成本绑定，这是为保持原型轻量化而做的简化。
2.6 公理的自足性与合法性定义
这五条公理构成一个闭合的公理系统。任何满足这些公理的构造，其内部运作天然合法，无需诉诸外部裁判。合法性定义为：
设公理集为 AA，事实集为 FF，能量预算为 EE，系统输出集合 OO 满足：
O⊆{ϕ∣A∪F⊢Eϕ}O⊆{ϕ∣A∪F⊢Eϕ}
其中 ⊢E⊢E 表示在能量预算 EE 内可推导。系统拒绝输出任何不在该推导闭包内的命题。
此定义不承诺外部世界的“绝对真”，仅保证系统内结论的可追溯性和形式一致性。

class Predicate:

        self.args = args
    def __repr__(self):
        return f"{self.name}({', '.join(str(a) for a in self.args)})"
    def __eq__(self, other):
        return (isinstance(other, Predicate) and self.name == other.name and
                len(self.args) == len(other.args) and
                all(a == b for a, b in zip(self.args, other.args)))
    def __hash__(self):
        return hash((self.name, tuple(self.args)))

from .term import Variable

def walk(term, theta):
    while isinstance(term, Variable) and term in theta:
        term = theta[term]
    return term

def occurs(var, term, theta):
    term = walk(term, theta)
    if var == term:
        return True
    if hasattr(term, 'args'):
        return any(occurs(var, arg, theta) for arg in term.args)
    return False

def unify(x, y, theta=None):
    if theta is None:
        theta = {}
    x = walk(x, theta)
    y = walk(y, theta)
    if x == y:
        return theta
    if isinstance(x, Variable):
        if occurs(x, y, theta):
            return None
        theta[x] = y
        return theta
    if isinstance(y, Variable):
        if occurs(y, x, theta):
            return None
        theta[y] = x
        return theta
    if hasattr(x, 'name') and hasattr(y, 'name') and x.name != y.name:
        return None
    if hasattr(x, 'args') and hasattr(y, 'args'):
        if len(x.args) != len(y.args):
            return None
        for a, b in zip(x.args, y.args):
            theta = unify(a, b, theta)
            if theta is None:
                return None
        return theta
    return None

from copy import deepcopy
from .unify import unify
from .term import Variable
from .predicate import Predicate
from .clause import Clause

class ProofNode:
    def __init__(self, goal, clause, theta, subproof):
        self.goal = goal
        self.clause = clause
        self.theta = theta
        self.subproof = subproof

class SLDEngine:
    def __init__(self, clauses, strategy, energy, tracer):
        self.clauses = clauses
        self.strategy = strategy
        self.energy = energy
        self.tracer = tracer
        self._fresh_id = 0

    def _fresh_var(self, base):
        self._fresh_id += 1
        return Variable(f"{base}_{self._fresh_id}")

    def _standardize_apart(self, clause):
        var_map = {}
        def rename(t):
            if isinstance(t, Variable):
                if t not in var_map:
                    var_map[t] = self._fresh_var(t.name)
                return var_map[t]
            elif hasattr(t, 'args'):
                new_args = [rename(arg) for arg in t.args]
                t.args = new_args
                return t
            else:
                return t
        new_head = rename(deepcopy(clause.head))
        new_body = [rename(deepcopy(g)) for g in clause.body]
        return Clause(new_head, new_body)

    def prove(self, goal):
        return self._sld([goal], 0)

    def _sld(self, goals, depth):
        if self.energy.remaining < 1:
            return False, None, 0
        if not goals:
            return True, [], 1
        first = goals[0]
        candidates = self.strategy.select_candidates(first, self.clauses)
        for clause in candidates:
            clause = self._standardize_apart(clause)
            theta = unify(first, clause.head, {})
            if theta is not None:
                new_body = [self._apply_subst(g, theta) for g in clause.body]
                new_goals = new_body + [self._apply_subst(g, theta) for g in goals[1:]]
                sub_success, sub_tree, sub_energy = self._sld(new_goals, depth+1)
                total_energy = 1 + sub_energy
                if sub_success:
                    node = ProofNode(first, clause, theta, sub_tree)
                    return True, node, total_energy
        return False, None, 1

    def _apply_subst(self, term, theta):
        if isinstance(term, Variable):
            return theta.get(term, term)
        elif hasattr(term, 'args'):
            new_args = [self._apply_subst(arg, theta) for arg in term.args]
            return Predicate(term.name, new_args)
        else:
            return term

class EnergyExhausted(Exception):
    pass

class EnergyLoop:
    def __init__(self, total=100):
        self.total = total
        self.remaining = total
    def consume(self, amount):
        if amount > self.remaining:
            raise EnergyExhausted(f"Need {amount}, only {self.remaining} left")
        self.remaining -= amount
    def can_continue(self, threshold=0):
        return self.remaining > threshold

import json
import time
import os

class TraceRecorder:
    def __init__(self, log_dir="logs"):
        self.log_dir = log_dir
        os.makedirs(log_dir, exist_ok=True)
        self.session_id = str(int(time.time()))
        self.steps = []
    def record_step(self, step, state, factor, energy, strategy, success, proof):
        step_data = {
            'step': step,
            'time': time.time(),
            'state': [str(s) for s in state] if state else [],
            'factor': factor,
            'energy_remaining': energy,
            'strategy': strategy,
            'success': success,
            'proof': str(proof) if proof else None
        }
        self.steps.append(step_data)
    def save(self, filename=None):
        if filename is None:
            filename = f"{self.log_dir}/trace_{self.session_id}.json"
        with open(filename, 'w') as f:
            json.dump(self.steps, f, indent=2)
        return filename

import hashlib
import json

class EvidenceNode:
    def __init__(self, node_type, content, source, timestamp, parents=None):
        self.type = node_type  # 'factor', 'evidence', 'conclusion'
        self.content = content
        self.source = source
        self.timestamp = timestamp
        self.parents = parents or []
        self.id = self._compute_hash()

    def _compute_hash(self):
        sorted_parents = sorted(self.parents)
        if sorted_parents:
            parent_combined = ''.join(sorted_parents)
            parent_hash = hashlib.sha256(parent_combined.encode()).hexdigest()
        else:
            parent_hash = ''
        data = {
            'type': self.type,
            'content': self.content,
            'source': self.source,
            'timestamp': self.timestamp,
            'parent_hash': parent_hash
        }
        return hashlib.sha256(json.dumps(data, sort_keys=True).encode()).hexdigest()

    def to_dict(self):
        return {
            'id': self.id,
            'type': self.type,
            'content': self.content,
            'source': self.source,
            'timestamp': self.timestamp,
            'parents': self.parents
        }

import networkx as nx
from .node import EvidenceNode

class EvidenceGraph:
    def __init__(self):
        self.graph = nx.DiGraph()
        self.nodes = {}

    def add_node(self, node):
        self.nodes[node.id] = node
        self.graph.add_node(node.id, **node.to_dict())

    def add_edge(self, from_id, to_id, relation):
        if from_id in self.nodes and to_id in self.nodes:
            self.graph.add_edge(from_id, to_id, relation=relation)

    def get_node(self, node_id):
        return self.nodes.get(node_id)

    def get_nodes_by_type(self, node_type):
        return [n for n in self.nodes.values() if n.type == node_type]

    def to_dict(self):
        return {
            'nodes': [n.to_dict() for n in self.nodes.values()],
            'edges': list(self.graph.edges(data=True))
        }

    @classmethod
    def from_dict(cls, data):
        g = cls()
        for nd in data['nodes']:
            node = EvidenceNode(
                nd['type'], nd['content'], nd['source'], nd['timestamp'],
                nd.get('parents', [])
            )
            node.id = nd['id']
            g.add_node(node)
        for u, v, attrs in data['edges']:

import json
import hashlib
import sqlite3
import time

class HeritageArchive:
    def __init__(self, db_path="heritage.db"):
        self.conn = sqlite3.connect(db_path)
        self.conn.execute('''CREATE TABLE IF NOT EXISTS heritage
                            (hash TEXT PRIMARY KEY,
                             prev_hash TEXT,
                             data TEXT,
                             timestamp REAL)''')
        self.conn.commit()

    def save(self, graph, conclusions, sentences, verification_log):
        cur = self.conn.execute("SELECT hash FROM heritage ORDER BY timestamp DESC LIMIT 1")
        row = cur.fetchone()
        prev_hash = row[0] if row else None

        data = {
            'graph': graph.to_dict(),
            'conclusions': conclusions,
            'sentences': sentences,
            'log': verification_log,
            'prev_hash': prev_hash,
            'timestamp': time.time()
        }
        data_str = json.dumps(data, sort_keys=True)
        h = hashlib.sha256(data_str.encode()).hexdigest()
        self.conn.execute("INSERT INTO heritage VALUES (?,?,?,?)",
                          (h, prev_hash, data_str, time.time()))
        self.conn.commit()
        return h

    def load(self, h):
        cur = self.conn.execute("SELECT data FROM heritage WHERE hash=?", (h,))
        row = cur.fetchone()
        if row:
            return json.loads(row[0])
        return None

        h = archive.save(graph, [], valid_sentences, verification_log)
        print(f"传承包哈希: {h}")

        energy.consume(1)

### Abstract

Large-scale statistical learning systems in practical applications often exhibit “hallucinations”—generating plausible yet factually incorrect statements in the absence of sufficient evidence. This paper argues that this issue is not merely a matter of insufficient engineering tuning but is intrinsically linked to the structural characteristics of the statistically dominant paradigm regarding verifiability and accountability.

We propose **Factor Recursion Theory (FRT)** as a formal axiomatic system. The system consists of five axioms: Real time Interaction, Structural Decoupling, Protocol Embedding, Computability & Traceability, and Power Efficiency. Unlike “generate and externally verify” patterns, this system imposes endogenous constraints on legitimate operations at the architectural level, ensuring that system outputs arise solely from reconstructible inference paths.

We further implement a lightweight logical inference engine prototype (v0.3.1), whose core is based on SLD resolution, supplemented by an energy consumption model and traceable recording structures. The demo code is publicly available, demonstrating the basic mechanism of “no evidence, no output”.

This paper does not claim to replace statistical methods; instead, it proposes a formal cognitive unit that can serve as a safety kernel for high reliability scenarios.

### Introduction

In recent years, deep learning models have achieved remarkable successes in natural language processing, computer vision, and other fields. However, in open environments, these models often exhibit:

- Fabrication of non existent facts;
- Construction of unverifiable citations;
- High confidence expressions in low confidence information contexts.

Although retrieval augmented generation (RAG), tool calling, and symbolic modules have been introduced to mitigate these issues, their core generation mechanisms still rely on statistical fitting functions. Statistical model outputs are not based on formal derivations but on conditional probability approximations of training distributions.

This paper contends that in scenarios requiring strong verifiability and accountability (e.g., legal assistance, mathematical proof, critical infrastructure control), the statistical paradigm suffers from a structural verifiability deficit. To address this, we construct a formal system where legitimacy is defined as: system outputs must belong to the closure of formal axioms and explicit, reconstructible inference chains.

---

### Part I: Structural Limitations of the Statistically Dominant Paradigm

#### 1.1 Theoretical Convergence vs. Reality

Statistical learning theory guarantees asymptotic consistency under ideal i.i.d. conditions with infinite samples. Real world systems, however, never satisfy these conditions: distributions drift, adversarial inputs exist, and samples are incomplete. Consequently, deviations between model outputs and physical reality are inevitable. This is not a theoretical flaw but an engineering norm.

#### 1.2 Lack of Explicit Verification

Current mainstream architectures lack explicit verification mechanisms for the reasoning process. A typical deep neural network is essentially a high dimensional function approximator; its internal representations are not directly interpretable, and its decision paths cannot be audited. Even if a model achieves 99.9% accuracy on a test set, the remaining 0.1% of errors can still be catastrophic in critical systems, and when errors occur it is impossible to trace the reasoning path or assign responsibility.

#### 1.3 Resource Assumptions and Physical Cost

Statistical models consume enormous computational resources during both training and inference, with virtually no physical constraints. Models can search and iterate indefinitely until they produce an answer that fits the statistical pattern. This “unlimited resources” assumption not only is uneconomical but also conceals the physical cost that cognitive processes ought to have—Landauer’s principle tells us that any information processing must pay an energy price. A cognitive system should embody resource constraints at the design level to be physically consistent.

#### 1.4 Contributions and Limitations of Hybrid Architectures

Hybrid architectures (RAG, tool calling, symbolic modules) have significantly improved reliability, but they often act as black box plugins: retrieval may fetch erroneous documents, tool calls may rely on unreliable APIs, symbolic constraints may fail in complex reasoning. More importantly, they lack a unified verifiable reasoning core; the overall decision process remains fragmented and cannot be traced end to end.

---

### Part II: Factor Recursion Theory – A Formal Axiomatic System

Factor Recursion Theory redefines the fundamental units of intelligent systems from an existential perspective; its five axioms constitute necessary and sufficient conditions for any system to survive over the long term. Unlike “generate and verify” patterns, these axioms are endogenous constraints embedded at design time—illegal operations are prevented at the architectural level.

#### 2.1 Axiom 1: Real time Interaction – Trace Anchoring

The system must maintain continuous, bidirectional real time interaction with the physical world. Interactions themselves leave traces, which constitute the physical anchors of cognition. In the current prototype, the knowledge base is static, but a dynamic update interface is reserved.

#### 2.2 Axiom 2: Structural Decoupling – Risk Isolation

Core modules such as survival, safety, allocation, and supervision must be structurally isolated. This prevents unverified outputs from the cognitive layer from propagating directly to the physical layer. In the inference engine, this is realised by a clear boundary between the inference core and external plugins.

#### 2.3 Axiom 3: Protocol Embedding – Rules Embedded Internally

All interactions involving rights, resources, or risks must be based on verifiable, accountable, and publicly published rules. At the reasoning level, every derivation must follow a formalised logical protocol (e.g., first order logic resolution). We adopt SLD resolution as the underlying logic protocol.

#### 2.4 Axiom 4: Computability & Traceability – Traceability

Key states, contributions, and costs must be observable, verifiable, and traceable. The state of each factor is the accumulation of historical interaction traces. The engine includes a built in TraceRecorder that logs the knowledge base, factors, energy, and decisions at every step.

#### 2.5 Axiom 5: Power Efficiency – Energy Closed Loop

The total matter energy information consumption per unit time must be below the environment's sustainable supply rate. In reasoning, every operation has a defined energy cost and the total energy is fixed. When energy is exhausted the process stops. In the current prototype, energy consumption is a simple counter ( +1 per step ) and is not yet tied to information theoretic costs; this simplification is intended to keep the prototype lightweight.

#### 2.6 Self Sufficiency of the Axioms and Definition of Legitimacy

These five axioms form a closed axiomatic system. Any construct that satisfies them is inherently legitimate in its internal operations, without recourse to an external judge. Legitimacy is defined as follows:

Let the axiom set be \(A\), the fact set be \(F\), and the energy budget be \(E\). The system output set \(O\) satisfies:
\[
O \subseteq \{ \phi \mid A \cup F \vdash_E \phi \}
\]
where \(\vdash_E\) denotes derivability within the energy budget \(E\). The system refuses any proposition outside this derivable closure.

This definition does not commit to “absolute truth” in the external world; it only guarantees traceability and formal consistency within the system.
